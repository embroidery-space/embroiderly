# Embroiderly v0.6.0

## New Features

### Reference Images

The main and biggest change in this release is the ability to place an image (PNG, JPEG, or WebP) under your pattern.
This lets you draw stitches based on the original image.

To add a reference image to your pattern, right-click on the working area and select **Add Reference Image** from the context menu.
The image will automatically adjust to fit your pattern dimensions.
Using the new **Cursor** tool, you can move and rotate the image, as well as change its size and transparency.

The image itself and all its settings are saved in the pattern project file (`.embproj`).

### Rulers

Rulers with markers for main lines now appear at the top and left of your pattern.
The markers indicate major grid intervals (for example, every ten stitches, depending on your pattern grid settings).
The rulers automatically adjust to the current zoom level and pattern position.

### Layer Visibility Settings

In the right panel, above the display mode switches, there's now a button with a layer visibility settings menu.
You can control the visibility of the reference image, each type of stitch, the grid, and the rulers.

### Zoom Controls

We've added a bottom panel to the working area that contains zoom controls.
You can change the zoom level using the slider, the nearby buttons, by entering a number, or by selecting an option from the dropdown menu.
You can also use the `Ctrl++` and `Ctrl+-` keyboard shortcuts.

### Improved Action History

Stitches drawn in one motion are now grouped in the action history as a single entry.
When you press `Ctrl+Z` or `Ctrl+Y` (or click the corresponding buttons in the application header, next to the settings button and window controls), the entire group of stitches will be undone or redone.

However, you can also undo or redo stitches one by one using `Ctrl+Shift+Z` and `Ctrl+Shift+Y` respectively.

### Telemetry Settings

We've integrated two services into the application:

- [Sentry](https://sentry.io/) for collecting error reports
- [PostHog](https://posthog.com/) for collecting application usage metrics

We use them to collect anonymous data about application usage.
This data helps us solve problems faster and analyze how you use the application.
It'll significantly help us improve Embroiderly.

Both services are disabled by default.
If you care about the development of our project, please enable them in the application settings under the **Telemetry** section.
Thank you!
