# Embroiderly v0.8.0

## Breaking Changes

### Pattern Format Change

Embroiderly v0.8.0 changes how `.embproj` files store your pattern internally.
Previous versions stored the pattern as `pattern.oxs` inside the project archive.
This version stores it as `pattern.json` in Embroiderly's own internal format instead.

**Patterns saved with earlier versions of Embroiderly won't open in v0.8.0.**

To recover a pattern from an older `.embproj` file:

1. Rename the file extension from `.embproj` to `.zip` and extract its contents (or use any archive manager that can open ZIP files directly).
2. Find the `pattern.oxs` file inside the archive.
3. Open that file in Embroiderly the same way you'd open any other external pattern.

### Linux: Bundled Chromium Embedded Framework

The Linux desktop installer now bundles the Chromium Embedded Framework (CEF) instead of relying on the system web driver.
This was necessary to support the new web capabilities, where the system web driver on Linux has limitations.

You'll notice the installer size is roughly 130 MB larger than before.
The update itself should install cleanly over a previous version.

## New Features

### Migrating to Web

Embroiderly is now available from any modern browser---no installation required.

Visit [embroiderly.niusia.me](https://embroiderly.niusia.me) on any device: desktop, tablet, or phone.
Windows, macOS, Linux, Android, and iOS are all supported.
You can also install Embroiderly as a [PWA](https://web.dev/articles/what-are-pwas) from your browser for an offline-capable, app-like experience.

Native desktop installers continue to be available for Windows and Linux.

### Custom Stitch Layers

You can now organize your pattern into multiple custom layers.
Each layer acts as an isolated drawing plane, so you can work on one part of your pattern without affecting the rest.
This is especially useful when refining a pattern after importing from an image---add a new layer, draw your corrections on top, and leave the original stitches untouched underneath.

Each custom layer automatically contains a set of stitch layers---one per stitch type.
You manage layers from the **Layers** section at the bottom of the canvas panel.

See the [Working with Patterns](../../guide/working-with-patterns#layers) chapter for full details.

### Native PDF Rendering

PDF export now draws pattern frames directly using native PDF graphics primitives instead of embedding SVG images.
The visual output is the same as before.

### Updated Grid Line Thickness

Grid line thickness is now a relative percentage rather than an absolute typographic point value.
This makes it easier to tweak the grid appearance without thinking about print resolution.

Also, a new **Pixel Line** option is introduced both for **Major Lines** and **Minor Lines**.
When enabled, the line is always drawn as a single pixel regardless of zoom level.

## Enhancements

- **Eraser tool.** A dedicated **Eraser** tool is now available in the toolbar.
  It removes any stitch regardless of type---useful on touch devices where `Alt+Click` isn't available.
- **Collapsible side panels.** The palette panel and canvas panel can now be collapsed to give the canvas more room.
  Use `Ctrl+Shift+L` to toggle the left panel and `Ctrl+Shift+R` to toggle the right panel.
- **Pattern tab indicators.** Pattern tabs now show a dot when a pattern has unsaved changes.
  The tabs themselves have moved to the application header to free up vertical space, which helps especially on small screens.
- **Open On Startup option.** A new setting controls what Embroiderly does at launch: open a **New Pattern** (default), open your **Custom Template**, or open **Nothing**.
- **Deleting special stitches.** Special stitches imported from Pattern Maker can now be deleted---both manually and as a result of resizing the pattern.
- **Intro tour.** New users see a guided tour that walks through adding a color to the working palette, drawing stitches, and adjusting display settings.
- **New demo patterns.** Several new demo patterns are now bundled with the app.
- **New and reorganized shortcuts.** Keyboard shortcut handling has been improved and new shortcuts have been added.
  See the updated [Shortcuts Reference](../../reference/shortcuts) chapter for the full list.
