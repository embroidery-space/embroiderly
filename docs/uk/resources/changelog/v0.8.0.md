# Embroiderly v0.8.0

## Кардинальні зміни

### Зміна формату схем

Embroiderly v0.8.0 змінює спосіб внутрішнього збереження схем у файлах `.embproj`.
Попередні версії зберігали схему як `pattern.oxs` всередині архіву проєкту.
Ця версія зберігає її як `pattern.json` у власному внутрішньому форматі Embroiderly.

**Схеми, збережені у попередніх версіях Embroiderly, не відкриватимуться у версії v0.8.0.**

Щоб відновити схему зі старого файлу `.embproj`:

1. Змініть розширення файлу з `.embproj` на `.zip` та розпакуйте його вміст (або скористайтеся будь-яким архіватором, що підтримує пряме відкриття файлів ZIP).
2. Знайдіть файл `pattern.oxs` всередині архіву.
3. Відкрийте цей файл в Embroiderly так само, як ви відкриваєте будь-яку іншу зовнішню схему.

### Linux: Вбудований Chromium Embedded Framework

Інсталятор для Linux тепер містить вбудований Chromium Embedded Framework (CEF) замість використання системного вебдрайвера.
Це було необхідно для підтримки нових вебможливостей, оскільки системний вебдрайвер у Linux має певні обмеження.

Ви помітите, що розмір інсталятора став приблизно на 130 МБ більшим, ніж раніше.
Саме оновлення має встановитися без проблем поверх попередньої версії.

## Нові функції

### Перехід у веб

Embroiderly тепер доступний у будь-якому сучасному браузері --- без необхідності встановлення.

Перейдіть за посиланням [embroiderly.niusia.me](https://embroiderly.niusia.me) з будь-якого пристрою: компʼютера, планшета чи телефона.
Підтримуються Windows, macOS, Linux, Android та iOS.
Ви також можете встановити Embroiderly як [PWA](https://web.dev/articles/what-are-pwas) з вашого браузера, щоб отримати досвід роботи офлайн у вигляді повноцінного застосунку.

Нативні інсталятори продовжують залишатися доступними для Windows та Linux.

### Власні шари схем

Тепер ви можете організувати свою схему за допомогою власних шарів.
Кожен шар працює як ізольована площина для малювання, тому ви можете працювати над однією частиною схеми, не впливаючи на інші.
Це особливо корисно при доопрацюванні схеми після імпорту з малюнка --- додайте новий шар та малюйте свої виправлення поверх нього, залишаючи оригінальні стібки недоторканими знизу.

Кожен власний шар автоматично містить набір шарів стібків --- по одному для кожного типу стібку.
Ви керуєте шарами в розділі **Шари** в нижній частині панелі полотна.

Детальнішу інформацію дивіться в розділі [Робота зі схемами](../../guide/working-with-patterns#шари).

### Нативний рендеринг PDF

Експорт PDF тепер малює схеми безпосередньо за допомогою нативних графічних примітивів PDF замість вбудовування зображень SVG.
Візуальний результат залишається таким самим, як і раніше.

### Оновлена товщина ліній сітки

Товщина ліній сітки тепер визначається у відсотках, а не в абсолютних типографічних пунктах.
Це спрощує налаштування вигляду сітки без необхідності замислюватися про роздільну здатність друку.

Також додано новий параметр **Піксельна лінія** для **Головних ліній** та **Малих ліній**.
Якщо його увімкнено, лінія завжди малюється товщиною в один піксель незалежно від рівня масштабування.

## Покращення

- **Інструмент Гумка.** На панелі інструментів тепер доступний окремий інструмент **Гумка**.
  Він видаляє будь-який стібок незалежно від його типу --- це корисно на сенсорних пристроях, де немає можливості використати комбінацію `Alt+ЛКМ`.
- **Згортні бічні панелі.** Панель палітри та панель полотна тепер можна згорнути, щоб звільнити більше місця для полотна.
  Використовуйте `Ctrl+Shift+L` для перемикання лівої панелі та `Ctrl+Shift+R` для перемикання правої панелі.
- **Індикатори вкладок схем.** Вкладки схем тепер відображають крапку, якщо схема має незбережені зміни.
  Самі вкладки перемістилися в заголовок застосунку, щоб звільнити вертикальний простір, що особливо корисно на маленьких екранах.
- **Опція "Відкривати під час запуску".** Нове налаштування визначає, що саме робить Embroiderly при запуску: відкриває **Нову схему** (типово), відкриває ваш **Власний шаблон** або не відкриває **Нічого**.
- **Видалення спеціальних стібків.** Спеціальні стібки, імпортовані з Pattern Maker, тепер можна видаляти --- як вручну, так і в результаті зміни розміру схеми.
- **Вступний тур.** Нові користувачі бачать інтерактивний тур, який допомагає додати колір до робочої палітри, намалювати стібки та налаштувати параметри відображення.
- **Нові демо-схеми.** Разом із застосунком тепер постачається кілька нових демо-схем.
- **Нові та переорганізовані сполучення клавіш.** Покращено обробку сполучень клавіш на клавіатурі та додано нові гарячі клавіші.
  Повний список дивіться в оновленому розділі [Сполучення клавіш](../../reference/shortcuts).
